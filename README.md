# SQL 기반 앱 분석 지표 프로젝트

## 프로젝트 개요

이벤트 로그 데이터를 활용하여 음식 배달 앱의 핵심 비즈니스 지표를 SQL로 정의하고 분석했습니다.

- **데이터**: Firebase/GA4 스타일 이벤트 로그
- **기술**: BigQuery SQL

## 주요 발견사항

### 1. 낮은 전환율과 핵심 이탈 지점

**전체 전환율: 21.7%**

- 총 52,823명 중 11,467명만 결제 완료
- 업계 평균 대비 낮은 수치

**3대 이탈 지점:**

| 순위 | 이탈 구간                    | 이탈률 | 이탈 인원 | 누적 영향 |
| ---- | ---------------------------- | ------ | --------- | --------- |
| 1위  | 음식점 선택 → 장바구니       | 38%    | 10,528명  | 매우 높음 |
| 2위  | 탐색(카테고리/검색) → 음식점 | 35%    | 14,812명  | 매우 높음 |
| 3위  | 장바구니 → 결제              | 34%    | 5,765명   | 높음      |

### 2. 리텐션 분석

**D1 Retention: 1-2%** <br/>

**해석:**

- 습관 형성 실패

## 핵심 문제 진단

### 문제 1: 음식점 선택 → 장바구니 이탈 (38%, 10,528명)

**현상:**

```
음식점 진입 → 메뉴 탐색 → 장바구니 담기 실패 → 이탈
```

**원인 (추정):**

- 음식점 리스트 매력도 부족
- 사진/리뷰 품질 낮음
- 필터링/정렬 옵션 부족

**개선안:**

- 음식점 사진 품질 개선
- 리뷰/별점 강조 표시
- 거리/배달시간 필터 추가
- 인기 음식점 우선 노출

### 문제 2: 탐색 → 음식점 이탈 (35%, 14,812명)

**현상:**

```
카테고리/검색/배너 탐색 → 음식점 선택 안 함 → 이탈
```

**원인 (추정):**

- 탐색 결과 관련성 부족
- 원하는 음식점을 찾지 못함

**개선안:**

- 검색 결과 개인화
- 카테고리별 인기 음식점 노출
- 배너 타겟팅 개선

### 문제 3: 장바구니 → 결제 이탈 (34%, 5,765명)

**현상:**

```
장바구니 추가 → 결제 안 함 → 이탈
```

**원인 (추정):**

- 배달비/최소주문금액 확인 후 포기
- 결제 과정 복잡
- 추가 쿠폰/할인 부족

**개선안:**

- 배달비 사전 안내 (메뉴 선택 단계)
- 간편결제 추가
- 최소주문금액 미달 시 추천 메뉴 제시
- 장바구니 이탈자 리타겟팅 푸시 알림
